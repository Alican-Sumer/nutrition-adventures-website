Nutrition Adventures: Code And System Explainer

This document explains how the game map is exported from Tiled, how Phaser loads it, how collision and trigger rectangles work, how Twine scenarios are embedded, and how player movement and animation are controlled.

Code base: 2026Spring-Team43-DHES/Game. The snippets below are copied from the current implementation.

1. System Overview

The main idea is simple: Tiled stores map data as JSON, Phaser reads that JSON, the game converts object layers into runtime rectangles, and those rectangles are used for blocking movement and launching scenarios.

2. How Tiled Exports The Map

The map file is assets/maps/demoMap_V3_3_08_26_json.json. This project uses:

  • A background image layer for the campus map art
  • A collisions object layer for blocked building areas
  • A triggers object layer for doors and interaction zones

This is important for your presentation: the game is not hardcoding the map geometry by hand. Instead, it relies on Tiled’s exported object layers.

3. How Phaser Loads The Tiled JSON

In src/scenes/Start.js, the game preloads both the background image and the Tiled JSON: 

preload() {
    this.load.image('demo-map-bg', 'assets/maps/NCSU_map_forDEMO_V2.png');
    this.load.tilemapTiledJSON('map', 'assets/maps/demoMap_V3_3_08_26_json.json');
    this.load.spritesheet('player-up', 'assets/sprite/idle_up.png', { frameWidth: 96, frameHeight: 80 });
    this.load.spritesheet('player-down', 'assets/sprite/idle_down.png', { frameWidth: 96, frameHeight: 80 });
    this.load.spritesheet('player-left', 'assets/sprite/idle_left.png', { frameWidth: 96, frameHeight: 80 });
    this.load.spritesheet('player-right', 'assets/sprite/idle_right.png', { frameWidth: 96, frameHeight: 80 });
}

Then the scene creates the map object and reads the raw JSON data: 

const map = this.make.tilemap({ key: 'map' });
const rawMapData = this.cache.tilemap.get('map').data;
const imageLayer = rawMapData.layers.find(
    (layer) => layer.name === 'background' && layer.type === 'imagelayer'
);

The code uses the background layer dimensions to compute the world bounds: 

const mapWidth = imageLayer.imagewidth || map.widthInPixels;
const mapHeight = imageLayer.imageheight || map.heightInPixels;
this.mapRect = new Phaser.Geom.Rectangle(bgX, bgY, mapWidth, mapHeight);
Phaser is using the JSON both as a content source and as a geometry source. The background image gives the visual world size, and the object layers give gameplay logic.

4. How Collision Rectangles Work

The collisions object layer from Tiled is converted into Phaser rectangles: 

this.collisionZones = this.createRectanglesFromObjectLayer(map, 'collisions');
createRectanglesFromObjectLayer(map, layerName) {
    const objectLayer = map.getObjectLayer(layerName);
    if (!objectLayer || !objectLayer.objects) {
        return [];
    }

    return objectLayer.objects
        .filter((obj) => typeof obj.x === 'number' && typeof obj.y === 'number')
        .map((obj) => new Phaser.Geom.Rectangle(obj.x, obj.y, obj.width || 0, obj.height || 0));
}

During movement, the player is treated as a circle. The scene checks whether that circle would intersect any collision rectangle before moving: 

tryMovePlayer(dx, dy) {
    const nextX = this.player.x + dx;
    const nextY = this.player.y + dy;
    const playerCircle = new Phaser.Geom.Circle(nextX, nextY, this.playerRadius);

    if (!Phaser.Geom.Rectangle.Contains(this.mapRect, nextX, nextY)) {
        return;
    }

    const hitsCollision = this.collisionZones.some((zone) =>
        Phaser.Geom.Intersects.CircleToRectangle(playerCircle, zone)
    );
    if (hitsCollision) {
        return;
    }

    this.player.setPosition(nextX, nextY);
}

So the logic is: compute where the player wants to go, check map bounds, check collision rectangles, and only then update the player position.

5. How Trigger Rectangles Launch Scenarios

The game reads the Tiled triggers object layer and stores each trigger as both a raw rectangle and a slightly enlarged activation rectangle: 

this.triggerZones = this.createTriggerZonesFromObjectLayer(map, 'triggers');
createTriggerZonesFromObjectLayer(map, layerName) {
    const objectLayer = map.getObjectLayer(layerName);
    if (!objectLayer || !objectLayer.objects) {
        return [];
    }

    return objectLayer.objects
        .filter((obj) => typeof obj.x === 'number' && typeof obj.y === 'number' && (obj.width || 0) > 0 && (obj.height || 0) > 0)
        .map((obj) => ({
            name: (obj.name || '').trim().toLowerCase(),
            type: (obj.type || '').trim().toLowerCase(),
            rect: new Phaser.Geom.Rectangle(obj.x, obj.y, obj.width || 0, obj.height || 0),
            activationRect: new Phaser.Geom.Rectangle(
                obj.x - this.triggerActivationHorizontalPadding,
                obj.y,
                (obj.width || 0) + this.triggerActivationHorizontalPadding * 2,
                (obj.height || 0) + this.triggerActivationForwardPadding
            )
        }));
}

Each trigger name from Tiled is then mapped to a scenario: 

this.triggerToScenario = {
    clark_door_trigger: 'scenario1',
    fountain_door_trigger: 'scenario2',
    case_door_trigger: 'scenario3',
    talley_door_trigger: 'scenario4',
    ut_door_trigger: 'scenario6',
    oval_door_trigger: 'scenario5'
};

The runtime trigger check happens here: 

checkTriggerScenarioActivation() {
    const triggerCircle = new Phaser.Geom.Circle(this.player.x, this.player.y, this.triggerActivationRadius);
    const hitZone = this.triggerZones.find((zone) =>
        Phaser.Geom.Intersects.CircleToRectangle(triggerCircle, zone.activationRect)
    );
    if (!hitZone) {
        this.activeTriggerScenario = null;
        return;
    }

    const triggerKey = hitZone.name || hitZone.type;
    const scenarioId = this.triggerToScenario[triggerKey];
    if (!scenarioId) {
        return;
    }

    if (this.activeTriggerScenario === scenarioId || this.twineOverlay.isOpen()) {
        return;
    }

    this.activeTriggerScenario = scenarioId;
    this.openScenario(scenarioId);
}
This is the core gameplay loop around dining halls: walking to a door places the player circle inside a trigger rectangle, and that rectangle launches a specific Twine lesson.

6. How Twine Scenarios Are Embedded

The game stores scenario metadata in the scene: 

this.scenarios = {
    scenario1: {
        title: 'Scenario 1',
        htmlPath: '/Scenarios/Scenario1/Scenario1.html',
        completionPassages: ['Key Takeaways']
    },
    scenario5: {
        title: 'Scenario 5',
        htmlPath: '/Scenarios/Scenario5/Scenario5.html',
        completionPassages: ['Finish']
    },
    scenario6: {
        title: 'Scenario 6',
        htmlPath: '/Scenarios/Scenario6/Scenario6.html',
        completionPassages: ['ScenarioEnd']
    }
};

When a scenario is opened, the scene delegates to TwineOverlay: 

openScenario(scenarioId) {
    const config = this.scenarios[scenarioId];
    if (!config) {
        return;
    }

    this.twineOverlay.show({
        scenarioId,
        title: config.title,
        htmlPath: config.htmlPath,
        completionPassages: config.completionPassages || [],
        onComplete: (completedScenarioId) => {
            this.player.completeScenario(completedScenarioId);
        }
    });
}

The overlay itself is a DOM-based full-screen panel with an iframe: 

show({ scenarioId, title, htmlPath, completionPassages = [], onComplete }) {
    this.ensureElements();
    this.currentScenarioId = scenarioId;
    this.completeCallback = onComplete;
    this.completionPassages = completionPassages;
    this.titleEl.textContent = title;
    this.iframeEl.src = htmlPath;
    this.container.style.display = 'flex';
    document.addEventListener('keydown', this.handleEsc);
    window.addEventListener('message', this.messageHandlerBound);
}
this.iframeEl = document.createElement('iframe');
this.iframeEl.style.border = '0';
this.iframeEl.style.width = '100%';
this.iframeEl.style.height = '100%';
this.iframeEl.style.background = '#f3ead8';
this.iframeEl.setAttribute('title', 'Twine Scenario');

The overlay detects completion in two ways:

  1. Twine can send postMessage({ type: 'twine:complete' })
  2. The bridge can watch SugarCube passage changes and detect specific ending passages
handleMessage(event) {
    if (event.source !== this.iframeEl?.contentWindow) {
        return;
    }

    const data = event.data;
    if (!data || typeof data !== 'object') {
        return;
    }

    if (data.type === 'twine:complete') {
        this.complete();
    }
}
jq(doc).on(':passageend.scenario-complete', () => {
    const currentPassage = win.SugarCube?.State?.passage;
    if (!currentPassage || !completionPassages.has(currentPassage)) {
        return;
    }

    win.parent.postMessage({
        type: 'twine:complete',
        scenarioId: this.currentScenarioId,
        passage: currentPassage
    }, '*');
});

When completion is detected, the overlay notifies both the local scene and the parent page: 

complete() {
    if (this.completeCallback && this.currentScenarioId) {
        this.completeCallback(this.currentScenarioId);
        window.parent.postMessage(
            { type: 'SCENARIO_COMPLETE', scenarioId: this.currentScenarioId }, '*'
        );
    }
    this.close();
}

7. How The Player Is Controlled

The scene owns input and movement math. The player class owns sprite state and animation.

In the scene, keyboard input is registered with Phaser: 

this.moveKeys = this.input.keyboard.addKeys('W,A,S,D');

Every frame, the scene computes an X/Y direction from those keys: 

const inputX = (this.moveKeys.D.isDown ? 1 : 0) - (this.moveKeys.A.isDown ? 1 : 0);
const inputY = (this.moveKeys.S.isDown ? 1 : 0) - (this.moveKeys.W.isDown ? 1 : 0);

If the player is moving, the input vector is normalized and scaled by speed: 

const vec = new Phaser.Math.Vector2(inputX, inputY)
    .normalize()
    .scale(this.playerSpeed * dt);

this.tryMovePlayer(vec.x, 0);
this.tryMovePlayer(0, vec.y);
this.checkTriggerScenarioActivation();

The scene also decides which direction the player should face: 

let direction = this.lastDirection;
if (Math.abs(inputX) > Math.abs(inputY)) {
    direction = inputX > 0 ? 'right' : 'left';
} else {
    direction = inputY > 0 ? 'down' : 'up';
}
this.lastDirection = direction;
this.player.setMovementDirection(direction, true);

In StudentPlayer.js, the player object handles animation switching: 

setMovementDirection(direction, moving) {
    if (!direction) {
        return;
    }

    if (this.currentDirection !== direction) {
        this.currentDirection = direction;
    }

    const animKey = `player-${this.currentDirection}-anim`;
    if (moving) {
        this.anims.play(animKey, true);
        return;
    }

    this.anims.stop();
    this.setTexture(`player-${this.currentDirection}`, 0);
}
This split is good design: the scene controls gameplay logic, while the player class controls how the sprite looks and animates.

8. How Phaser Starts The Whole Game

The Phaser entry point is src/main.js: 

import { Intro } from './scenes/Intro.js';
import { Start } from './scenes/Start.js';

const config = {
    type: Phaser.AUTO,
    title: 'Nutrition Adventures',
    parent: 'game-container',
    width: 1024,
    height: 1024,
    scene: [
        Intro,
        Start
    ],
    scale: {
        mode: Phaser.Scale.FIT,
        autoCenter: Phaser.Scale.CENTER_BOTH
    },
}

new Phaser.Game(config);

So the boot flow is:

  1. Phaser creates the game instance
  2. The intro scene loads first
  3. The start scene loads the map, player, collisions, and triggers
  4. Walking to doors launches Twine scenarios in the overlay

9. How Scenario Files Are Served

The game server is custom because the scenarios live outside the main Game folder, in a sibling top-level Scenarios directory. 

const gameRootDir = path.resolve(__dirname, '..');
const repoRootDir = path.resolve(gameRootDir, '..');
const scenariosRootDir = path.join(repoRootDir, 'Scenarios');
const baseDir = requested.startsWith('/Scenarios/')
    ? scenariosRootDir
    : gameRootDir;

That means a URL like /Scenarios/Scenario5/Scenario5.html is routed to the real scenario folder on disk. Without that server logic, the Twine files would not load correctly.

10. Short Presentation Script

If you need a short speaking version:

  1. We built the campus map in Tiled and exported it as JSON.
  2. The JSON includes a background image layer plus object layers for collisions and triggers.
  3. Phaser reads those object layers and turns them into runtime rectangles.
  4. Collision rectangles stop the player from walking through buildings.
  5. Trigger rectangles placed at dining hall doors launch Twine-based nutrition scenarios.
  6. The player is controlled with WASD, and the scene handles movement while the player class handles animation.
  7. Twine runs inside an iframe overlay, and completion is detected through passage endings or messages back to the game.
